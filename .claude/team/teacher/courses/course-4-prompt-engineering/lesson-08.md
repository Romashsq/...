# Урок 8: Управление форматом вывода — списки, таблицы, JSON, Markdown

**Модуль:** 2 — Фреймворки
**Курс:** Промпт-инженерия
**Время:** ~20 минут

---

## Введение

Один и тот же ответ в разном формате — это разная ценность. Таблица с конкурентами которую можно сразу вставить в презентацию стоит больше чем сплошной текст про тех же конкурентов. Управление форматом — это навык превращать ответы ИИ в готовые к использованию артефакты.

---

## Почему формат важен

Представь: ты просишь ИИ сравнить 5 инструментов автоматизации. Без указания формата ты получишь 800 слов сплошного текста который нужно читать 10 минут и потом переформатировать. С форматом "таблица 5×4" — ты получаешь готовую сравнительную матрицу за секунды.

Формат = разница между "нужно переработать" и "готово к использованию".

---

## Основные форматы и когда их использовать

### Список (маркированный или нумерованный)

**Когда:** Перечисление идей, шагов, рекомендаций, вариантов.

```
Формат: нумерованный список из 5 пунктов.
Каждый пункт: заголовок (жирным) + 1-2 предложения объяснения.
```

**Пример результата:**
1. **Чёткий заголовок** — одна мысль, без воды
2. **Конкретный совет** — действие, а не наблюдение

### Таблица

**Когда:** Сравнение вариантов, анализ по критериям, матрицы решений.

```
Формат: таблица Markdown.
Строки: [объекты сравнения].
Столбцы: [критерии].
Последняя строка: итог/рекомендация.
```

**Пример запроса:**
```
Сравни Make.com, Zapier и n8n.
Формат: таблица. Строки: инструменты. Столбцы: цена, простота, возможности, поддержка ИИ.
Оценка каждого: 1-5 звёзд.
```

### JSON

**Когда:** Структурированные данные для дальнейшей обработки, экспорта, или использования в коде.

```
Формат: JSON.
Структура: массив объектов, каждый объект содержит поля: name, description, price, pros, cons.
```

**Пример:** Генерация данных для базы знаний, контента для сайта, структуры уроков.

### Markdown

**Когда:** Документы, статьи, README файлы, контент для платформ которые рендерят Markdown.

```
Формат: Markdown.
Структура: H2 заголовки для разделов, H3 для подразделов.
Жирным: ключевые термины. Курсивом: примеры.
Код: в блоках ```
```

### Сплошной текст

**Когда:** Продающие тексты, email, посты для соцсетей, нарративные объяснения.

```
Формат: связный текст, 3 абзаца.
Стиль: разговорный, от первого лица.
Без заголовков и маркеров.
```

---

## Продвинутые техники форматирования

### Вложенные структуры

```
Формат: список из 3 стратегий.
Каждая стратегия:
  - Название (H3)
  - Описание (2 предложения)
  - Пример из реального бизнеса
  - Метрика для отслеживания результата
```

### Шаблон с заполнителями

```
Создай шаблон email с заполнителями в квадратных скобках:
[ИМЯ_КЛИЕНТА], [НАЗВАНИЕ_ПРОДУКТА], [КОНКРЕТНАЯ_ВЫГОДА].
```
Это создаёт переиспользуемый шаблон.

### Несколько вариантов в одном ответе

```
Дай 3 варианта заголовка.
Формат каждого варианта:
  Вариант [N]: "[текст заголовка]"
  Стратегия: [1 предложение почему это работает]
  Для кого: [тип аудитории]
```

## Примеры с разными форматами

**Задача:** Проанализировать целевую аудиторию курса.

❌ Без формата: получишь сплошной текст на 500 слов.

✅ С форматом:
```
Проанализируй целевую аудиторию онлайн-курса по промпт-инженерии для маркетологов.
Формат: 3 персоны покупателя.
Для каждой персоны:
  - Имя и возраст
  - Должность и компания
  - Главная боль (1 предложение)
  - Что хочет получить от курса
  - Главное возражение перед покупкой
  - Какой триггер заставит купить
```

## Практическое задание

Возьми любой длинный ответ который ты получал от ИИ и который было сложно использовать напрямую. Перепиши промпт добавив конкретный блок "Формат:" с:
- Типом вывода (список/таблица/JSON/текст)
- Структурой (из чего состоит каждый элемент)
- Длиной или количеством

Отправь новую версию. Сравни насколько результат стал более применимым.

## Тест

**Вопрос 1:** Когда использовать формат "таблица"?

A) Всегда — это самый удобный формат
B) Когда нужно сравнить несколько вариантов по одним критериям
C) Только для числовых данных
D) Когда текст слишком длинный

**Вопрос 2:** Ты создаёшь контент для сайта который рендерит Markdown. Какой формат запросить?

A) JSON
B) Сплошной текст
C) Markdown с H2/H3 заголовками и блоками кода
D) HTML

**Вопрос 3:** Для чего удобен JSON формат ответа ИИ?

A) Для чтения людьми — он красиво выглядит
B) Для структурированных данных которые будут использоваться в коде или экспортироваться
C) Это самый краткий формат
D) JSON обязателен для технических задач

**Вопрос 4:** Что значит "шаблон с заполнителями"?

A) Шаблон в котором ИИ оставляет пустые строки
B) Текст с метками [ТАК] которые можно заменить на конкретные значения
C) Формат для программистов
D) Стандартный ответ без персонализации

**Вопрос 5:** Ты хочешь email для рассылки — живой, от первого лица, без структуры. Какой формат указать?

A) JSON
B) Таблица
C) Markdown с заголовками
D) Связный текст 3 абзаца, разговорный стиль, без маркеров

---

### Ответы:

1. **B** — Таблица идеальна для сравнений. Строки = объекты, столбцы = критерии. Одним взглядом видишь всю картину.
2. **C** — Markdown с заголовками H2/H3 и блоками кода будет правильно отрендерен на платформе. Сплошной текст потеряет структуру.
3. **B** — JSON это машиночитаемый формат. Удобен когда данные нужно использовать в коде, вставить в базу данных или обработать автоматически.
4. **B** — Заполнители [ИМЯ], [ПРОДУКТ] превращают текст в переиспользуемый шаблон. Меняешь заполнитель — получаешь персонализированный вариант.
5. **D** — Для email-рассылки нужен живой текст без структурных элементов. Явно укажи: связный текст, от первого лица, без маркеров и заголовков.
