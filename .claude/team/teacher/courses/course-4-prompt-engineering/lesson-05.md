# Урок 5: Фреймворк RCTF — Role, Context, Task, Format

**Модуль:** 2 — Фреймворки
**Курс:** Промпт-инженерия
**Время:** ~20 минут

---

## Введение

Фреймворки — это не ограничения. Это скелет, который освобождает тебя от необходимости каждый раз думать "что ещё добавить в промпт?". RCTF — один из самых простых и мощных фреймворков. Четыре буквы, которые изменят качество твоих результатов навсегда.

---

## Что такое RCTF

**R — Role (Роль):** Кем является ИИ в этом взаимодействии?
**C — Context (Контекст):** Что ИИ должен знать о ситуации?
**T — Task (Задача):** Что конкретно нужно сделать?
**F — Format (Формат):** Как должен выглядеть результат?

Запомни как слово: **Р-К-Т-Ф**. Каждый раз когда пишешь промпт — мысленно проходи по этим четырём буквам.

---

## Разбор каждого компонента

### R — Role (Роль)

Роль — это не просто "притворись кем-то". Это инструкция для модели о том, какой угол зрения применять и какие знания активировать.

Слабая роль: "Ты — помощник"
Средняя роль: "Ты — маркетолог"
Сильная роль: "Ты — директор по маркетингу EdTech стартапа с опытом запуска 3 курсов и аудиторией в Восточной Европе"

Чем конкретнее роль — тем точнее ответ. Добавляй:
- Специализацию
- Опыт (лет, количество проектов)
- Контекст работы (индустрия, регион, тип компании)

### C — Context (Контекст)

Контекст отвечает на вопросы: кто ты, что происходит, что уже сделано, какие есть ограничения.

Хороший контекст включает:
- О себе: кто ты, чем занимаешься
- О задаче: почему это важно, какова конечная цель
- О том что уже есть: что пробовал, что не сработало
- Ограничения: бюджет, время, размер аудитории

Пример слабого контекста: "У меня бизнес"
Пример сильного контекста: "Я запускаю онлайн-курс по промпт-инженерии для маркетологов. Уже есть 58 уроков, платформа работает, но нет ни одного платного студента. Бюджет на маркетинг: $500. Аудитория: специалисты в Украине и диаспоре."

### T — Task (Задача)

Задача должна быть:
- Одна (не "напиши и придумай и проанализируй")
- Конкретная (не "помоги с маркетингом")
- С измеримым результатом (5 идей, 200 слов, 3 варианта)

Формула: Глагол действия + что именно + для чего

Пример: "Напиши (глагол) 5 заголовков для лендинга (что) которые увеличат конверсию регистраций (для чего)"

### F — Format (Формат)

Формат избавляет тебя от постобработки. Если ты скажешь что хочешь получить — ты получишь это сразу.

Что можно указать в формате:
- Структура: "заголовок + 3 пункта + CTA"
- Длина: "до 200 слов" или "1 страница А4"
- Стиль: "разговорный", "экспертный", "дружелюбный"
- Тип: "список", "таблица", "сплошной текст", "JSON"
- Язык: "русский", "простой язык без жаргона"

## Примеры RCTF в действии

**Пример 1: Создание контента**

```
R: Ты — опытный SMM-специалист с фокусом на EdTech и аудиторией в СНГ.

C: Я запускаю курс по промпт-инженерии для маркетологов без технического бэкграунда.
Цена: $49. Старт через 2 недели. Telegram-канал: 800 подписчиков.

T: Напиши 3 поста для Telegram анонсирующих запуск курса.
Каждый пост — разный угол: боль аудитории / результат / социальное доказательство.

F: Каждый пост: 100-150 слов. Эмодзи: 2-3 штуки.
Заканчивается вопросом или призывом к действию.
```

**Пример 2: Анализ**

```
R: Ты — аналитик рынка EdTech с опытом оценки онлайн-курсов.

C: Мой курс по промпт-инженерии конкурирует с Coursera AI, Skillbox и локальными
телеграм-каналами. Цена $49. Формат: самостоятельное обучение + AI-ассистент.

T: Проведи конкурентный анализ по 4 критериям:
цена, глубина контента, практика, поддержка студентов.

F: Таблица: конкуренты в строках, критерии в столбцах.
Оценка каждого критерия: 1-10.
После таблицы: 2-3 предложения с главным выводом.
```

---

## Шаблон RCTF (сохрани себе)

```
R: Ты — [специализация] с [опытом/контекстом].

C: [Кто ты и чем занимаешься — 2-3 предложения].
[Что важно знать об этой задаче — 1-2 предложения].

T: [Глагол действия] + [конкретно что] + [для чего/с каким результатом].

F: [Структура результата].
[Длина/объём].
[Стиль и тон].
```

---

## Практическое задание

Выбери одну реальную задачу из своей текущей работы. Напиши промпт по шаблону RCTF:

1. Открой текстовый редактор (или Notion)
2. Напиши заголовки: **R:**, **C:**, **T:**, **F:**
3. Заполни каждый блок
4. Отправь в Claude или ChatGPT
5. Оцени результат по шкале 1-10

Потом удали из промпта блок C (контекст) и отправь снова. Сравни результаты — это наглядно покажет как сильно контекст влияет на качество.

## Тест

**Вопрос 1:** Что означает буква C в RCTF?

A) Character — характер персонажа ИИ
B) Context — контекст ситуации и задачи
C) Command — команда для выполнения
D) Creativity — уровень креативности ответа

**Вопрос 2:** Какая из формулировок задачи самая сильная?

A) "Помоги с email-маркетингом"
B) "Напиши email"
C) "Напиши welcome-письмо для новых студентов курса, цель — побудить начать первый урок в течение 24 часов"
D) "Сделай что-нибудь с email-рассылкой"

**Вопрос 3:** Зачем указывать формат "таблица" вместо "список"?

A) Таблица — единственный формат который понимает ИИ
B) Таблица позволяет сравнивать несколько вариантов по одним критериям одновременно
C) Список ИИ делает плохо
D) Это просто личные предпочтения, разницы нет

**Вопрос 4:** Какой блок RCTF чаще всего пропускают новички?

A) R — роль
B) C — контекст
C) T — задача
D) F — формат

**Вопрос 5:** Ты хочешь ИИ написал текст "как для умных людей без воды". Куда это добавить?

A) В блок R — это часть роли
B) В блок T — это часть задачи
C) В блок F — это требование к формату и стилю
D) Не нужно — ИИ сам знает как писать

---

### Ответы:

1. **B** — Context. Контекст — это вся информация о ситуации которую ИИ не может знать сам.
2. **C** — Есть глагол (напиши), конкретный объект (welcome-письмо для новых студентов) и цель (побудить начать урок за 24ч).
3. **B** — Таблица идеальна для сравнения. Строки = объекты, столбцы = критерии. Одним взглядом видишь всю картину.
4. **B** — Контекст кажется очевидным автору, но ИИ ничего не знает о твоей ситуации. Это самый ценный и самый пропускаемый блок.
5. **C** — Требования к стилю, тону и уровню языка идут в блок Format.
