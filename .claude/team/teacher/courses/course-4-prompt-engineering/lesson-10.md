# Урок 10: Few-Shot Prompting — учи ИИ на примерах

**Модуль:** 3 — Продвинутые техники
**Курс:** Промпт-инженерия
**Время:** ~20 минут

---

## Введение

Есть способ научить ИИ делать именно то что ты хочешь — без длинных объяснений. Просто покажи несколько примеров. Это называется Few-Shot Prompting, и это одна из самых мощных техник в арсенале промпт-инженера. После этого урока ты перестанешь описывать что хочешь и начнёшь показывать.

---

## Что такое Few-Shot Prompting

Few-Shot — это когда ты даёшь ИИ несколько примеров "вход → выход" перед основным запросом. Модель видит паттерн и применяет его к новому запросу.

- **Zero-Shot:** просто задача, без примеров
- **One-Shot:** один пример + задача
- **Few-Shot:** 2-5 примеров + задача

Чем сложнее паттерн — тем больше примеров нужно. Для стиля письма достаточно 1-2. Для сложной классификации — 3-5.

---

## Как это работает

Формула:
```
[Объяснение задачи]

Примеры:
Вход: [пример 1]
Выход: [результат 1]

Вход: [пример 2]
Выход: [результат 2]

Теперь:
Вход: [твоя реальная задача]
Выход:
```

Модель анализирует паттерн из примеров и продолжает его для твоего запроса.

## Когда Few-Shot незаменим

**1. Стиль и тон**
Слова не передают голос так точно как примеры.

```

Мой стиль: "Перестань гадать. Начни делать."

Плохой заголовок: "5 советов для успеха"
Мой стиль: "5 вещей которые я бы сделал иначе если бы начинал заново"

Теперь перепиши:
Плохой заголовок: "Как начать использовать ИИ в работе"
Мой стиль:
```

**2. Классификация**
```
Классифицируй отзывы студентов по категориям.

Примеры:
Отзыв: "Контент отличный, но платформа тормозит"
Категория: технические проблемы + позитивный контент

Отзыв: "Хочу больше практических заданий"
Категория: запрос на функционал

Отзыв: "Не могу войти в аккаунт второй день"
Категория: критическая техническая проблема

Теперь классифицируй:
Отзыв: "Уроки понятные, но квизы слишком лёгкие"
Категория:
```

**3. Генерация в заданном формате**
```
Генерируй описания уроков по этому шаблону.

Пример 1:
Тема: Chain-of-Thought
Описание: Узнаешь как заставить ИИ думать вслух пошагово — это улучшает точность сложных задач на 40%. Практика: решишь 3 задачи используя эту технику.

Пример 2:
Тема: Role Prompting
Описание: Научишься активировать нужную экспертизу через правильную роль. Одна фраза меняет качество ответа кардинально. Практика: протестируешь 3 разных роли на одной задаче.

Теперь напиши описание для темы: Few-Shot Prompting
Описание:
```

---

## Сколько примеров нужно

| Задача | Примеров |
|---|---|
| Простой стиль/тон | 1-2 |
| Сложный стиль с нюансами | 3-4 |
| Классификация (мало категорий) | 2-3 |
| Классификация (много категорий) | 4-6 |
| Сложный формат | 2-3 |

Больше 5-6 примеров редко нужно — эффект убывает.

---

## Практическое задание

Возьми любой повторяющийся тип контента который ты создаёшь (посты, email, описания, заголовки). Найди 2-3 примера которые тебе нравятся. Составь Few-Shot промпт по формуле выше. Сгенерируй 5 новых вариантов. Оцени насколько точнее результат по сравнению с обычным промптом.

## Тест

**Вопрос 1:** Чем Few-Shot отличается от Zero-Shot?

A) Few-Shot используется только для сложных задач
B) Few-Shot включает 2-5 примеров "вход→выход" перед основным запросом
C) Few-Shot требует платного плана
D) Few-Shot работает только на английском языке

**Вопрос 2:** Для какой задачи Few-Shot наиболее полезен?

A) Перевод текста на другой язык
B) Передача конкретного стиля или голоса бренда
C) Получение фактической информации
D) Написание кода

**Вопрос 3:** Сколько примеров обычно достаточно для стиля письма?

A) 10+ для надёжности
B) 1-2 примера
C) Примеры не нужны для стиля
D) Минимум 7

**Вопрос 4:** Как работает Few-Shot технически?

A) ИИ запоминает твои примеры навсегда
B) Модель видит паттерн входа/выхода и применяет его к новому запросу
C) Примеры загружаются в базу данных ИИ
D) Примеры блокируют нежелательные ответы

**Вопрос 5:** Где в промпте должны стоять примеры?

A) После основного запроса
B) В отдельном чате
C) До основного запроса — сначала примеры, потом задача
D) Порядок не важен

---

### Ответы:
1. **B** — Few-Shot = несколько примеров перед задачей. Zero-Shot = только задача без примеров.
2. **B** — Стиль и голос невозможно точно передать словами. Примеры показывают паттерн лучше любого описания.
3. **B** — Для стиля 1-2 примера обычно достаточно. Больше примеров нужно для сложных паттернов классификации.
4. **B** — Модель анализирует паттерн из примеров и экстраполирует его на новый запрос.
5. **C** — Примеры должны идти до задачи — модель читает слева направо и строит паттерн из того что видела раньше.
