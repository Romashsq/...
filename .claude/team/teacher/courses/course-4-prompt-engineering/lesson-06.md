# Урок 6: Фреймворк APE — Action, Purpose, Expectation

**Модуль:** 2 — Фреймворки
**Курс:** Промпт-инженерия
**Время:** ~20 минут

---

## Введение

RCTF — мощный фреймворк для сложных задач. Но иногда нужен более компактный подход. APE — это трёхшаговая формула которая работает быстро и даёт чёткие результаты. Особенно хорошо подходит для задач где главное — это конкретное действие и ожидаемый результат.

---

## Что такое APE

**A — Action (Действие):** Что именно нужно сделать? Глагол + объект.
**P — Purpose (Цель):** Зачем это нужно? Какова конечная цель?
**E — Expectation (Ожидание):** Что ты ожидаешь получить? Формат, длина, критерии.

APE отличается от RCTF тем, что фокусируется на результате, а не на контексте. Это делает его быстрее — когда у тебя нет времени писать полный контекст, но нужно получить качественный ответ.

---

## Как работает каждый компонент

### A — Action

Action — это не просто задача. Это конкретное действие с чётким объектом.

Слабый Action: "Помоги мне"
Средний Action: "Напиши текст"
Сильный Action: "Напиши серию из 5 email-писем для новых студентов онлайн-курса"

Формула Action: **Глагол + Объект + Уточнение**

Лучшие глаголы для Action:
- Напиши, создай, сгенерируй — создание контента
- Проанализируй, оцени, сравни — анализ
- Придумай, предложи, перечисли — брейнсторм
- Переработай, улучши, сократи — редактура
- Объясни, упрости, разбери — объяснение

### P — Purpose

Purpose отвечает на вопрос "почему это важно". Этот блок часто пропускают — и зря. Именно Purpose заставляет ИИ выбирать правильные аргументы, тон и содержание.

Без Purpose: ИИ выполняет задачу механически.
С Purpose: ИИ понимает конечную цель и оптимизирует под неё.

Пример:
- Без: "Напиши email"
- С Purpose: "Напиши email. Цель — вернуть студента который не заходил на платформу 7 дней."

Когда ИИ знает цель, он сам выберет правильный тон (заботливый, не навязчивый), правильные аргументы (напомнит о прогрессе, предложит помощь) и правильный CTA (не "купи снова", а "продолжи где остановился").

### E — Expectation

Expectation — это твои ожидания от результата. Что должно получиться?

Включай:
- Формат (письмо, список, таблица, план)
- Длина или объём
- Стиль и тон
- Конкретные требования (без шаблонных фраз, только факты, с примерами)

---

## Сравнение RCTF и APE

| | RCTF | APE |
|---|---|---|
| Лучше для | Сложных задач с контекстом | Быстрых задач с фокусом на результате |
| Скорость написания | Медленнее | Быстрее |
| Глубина контекста | Высокая | Средняя |
| Когда использовать | Важный контент, специфический бизнес | Стандартные задачи, быстрые запросы |

## Примеры APE

**Пример 1: Email-кампания**

```
A: Напиши 3 варианта темы письма для email-рассылки по базе 500 подписчиков

P: Письмо анонсирует старт курса по промпт-инженерии.
Цель темы — добиться open rate выше 30%.

E: Каждый вариант: до 50 символов, без слова "бесплатно".
Форматы: вопрос / цифра + выгода / интрига.
После каждого варианта: 1 предложение — почему он сработает.
```

**Пример 2: Презентация**

```
A: Создай структуру презентации о промпт-инженерии для корпоративного тренинга

P: Тренинг для отдела маркетинга крупной компании, 20 человек, 2 часа.
Цель: чтобы после тренинга участники начали использовать ИИ в работе.

E: 8-10 слайдов. Для каждого слайда: заголовок + 3-4 ключевых тезиса.
Последний слайд — домашнее задание для практики.
```

**Пример 3: Быстрый контент**

```
A: Напиши 10 идей для постов в LinkedIn о промпт-инженерии

P: Позиционировать меня как эксперта в теме AI для бизнеса.

E: Каждая идея: 1 строка. Разные форматы: кейс / урок / провокационный вопрос / список.
```

---

## Когда APE лучше RCTF

Используй APE когда:
- Задача стандартная (email, пост, список)
- Мало времени на написание промпта
- Контекст понятен из самого действия
- Нужно быстро получить несколько вариантов

Используй RCTF когда:
- Задача специфическая для твоего бизнеса
- ИИ не может знать твою аудиторию без объяснения
- Важный контент (лендинг, питч, стратегия)
- Первый раз работаешь с новой темой

## Практическое задание

Напиши три промпта по формуле APE для реальных задач из твоей работы прямо сейчас:

**Задача 1:** Что-то что нужно написать (пост, письмо, текст)
**Задача 2:** Что-то что нужно проанализировать или оценить
**Задача 3:** Что-то для брейнсторма (идеи, названия, концепции)

Для каждой задачи напиши A, P и E — по одному-двум предложениям. Отправь в ИИ и сравни насколько быстро получился применимый результат по сравнению с твоими обычными запросами.

## Тест

**Вопрос 1:** Что означает P в APE?

A) Prompt — сам промпт
B) Purpose — цель и смысл задачи
C) Parameters — параметры настройки
D) Profile — профиль пользователя

**Вопрос 2:** Почему Purpose важен даже для простых задач?

A) Это обязательное требование платформы
B) ИИ не выполнит задачу без цели
C) Зная цель, ИИ оптимизирует содержание под конечный результат, а не просто выполняет механически
D) Purpose увеличивает длину ответа

**Вопрос 3:** Для какой задачи лучше подойдёт APE чем RCTF?

A) Написание стратегии на год для специфической ниши
B) Быстрая генерация 10 идей для постов
C) Анализ конкурентов с учётом специфики бизнеса
D) Создание кастомного онбординга для новых сотрудников

**Вопрос 4:** Ты пишешь APE промпт для анализа. Какой глагол использовать в блоке Action?

A) "Помоги с анализом"
B) "Сделай анализ"
C) "Проанализируй" + конкретный объект
D) "Думай об анализе"

**Вопрос 5:** В блоке Expectation ты указал "тон дружелюбный". Что ещё стоит добавить?

A) Ничего — тон это всё что важно
B) Формат результата и длину/объём
C) Роль ИИ
D) Историю компании

---

### Ответы:

1. **B** — Purpose = цель. Это ответ на вопрос "зачем мне этот результат" — что ИИ должен оптимизировать.
2. **C** — Email с целью "продать" и email с целью "вернуть студента" — это разные письма. Цель определяет аргументы, тон и призыв к действию.
3. **B** — Быстрая генерация идей не требует глубокого контекста. APE справится быстрее и даст нужный результат.
4. **C** — Конкретный глагол + объект = чёткая задача. "Проанализируй воронку продаж за последний квартал" — правильная формула Action.
5. **B** — Тон без формата и объёма оставляет ИИ свободу угадывать. Укажи: список или таблица, сколько пунктов, какая длина.
