# Урок 11: Chain-of-Thought — заставь ИИ думать пошагово

**Модуль:** 3 — Продвинутые техники
**Курс:** Промпт-инженерия
**Время:** ~20 минут

---

## Введение

Ты когда-нибудь замечал что ИИ уверенно даёт неправильный ответ на задачу с несколькими шагами? Это происходит когда модель "прыгает" к ответу без промежуточных рассуждений. Chain-of-Thought — техника которая заставляет ИИ думать вслух и пошагово. Результат: точность сложных задач вырастает в разы.

---

## Что такое Chain-of-Thought

Chain-of-Thought (CoT) — это когда ты просишь ИИ показать ход рассуждений перед финальным ответом. Модель не просто выдаёт вывод — она проходит через логические шаги которые к нему ведут.

Это особенно важно для:
- Математических и логических задач
- Многошаговых анализов
- Принятия решений с несколькими факторами
- Задач где важна логика, а не только факты

---

## Три способа активировать CoT

### Способ 1: Прямой запрос
```
Реши эту задачу шаг за шагом:
[задача]
```

### Способ 2: Мета-инструкция
```
Прежде чем дать ответ, проанализируй задачу.
Покажи своё рассуждение, потом дай финальный вывод.
```

### Способ 3: Структурированный CoT
```
Ответь на вопрос используя следующий формат:
1. Определи ключевые факторы
2. Проанализируй каждый фактор
3. Взвесь за и против
4. Дай обоснованный вывод
```

---

## Примеры применения

**Пример 1: Бизнес-решение**

❌ Без CoT:
```
Стоит ли мне нанять SMM-менеджера или отдать на аутсорс?
```
Получишь: общий ответ "зависит от ситуации, вот плюсы и минусы..."

✅ С CoT:
```
Помоги принять решение: нанять SMM-менеджера (зарплата $800/мес) или
аутсорс агентство ($400/мес).

Контекст: стартап, 5 человек, 2000 подписчиков в Telegram,
цель — вырасти до 10К за 6 месяцев.

Думай пошагово:
1. Сначала определи что важнее для нашей цели: скорость/контроль/стоимость
2. Оцени каждый вариант по этим критериям
3. Учти риски каждого варианта
4. Дай итоговую рекомендацию с обоснованием
```

**Пример 2: Анализ текста**

```
Проанализируй этот лендинг и найди слабые места.

Думай как опытный CRO-специалист:
Шаг 1: Оцени первый экран — понятно ли что предлагается за 3 секунды?
Шаг 2: Проверь ценностное предложение — конкретное или общее?
Шаг 3: Найди точки трения — где пользователь может уйти?
Шаг 4: Оцени CTA — понятно ли что произойдёт после клика?
Шаг 5: Дай топ-3 изменения с наибольшим влиянием на конверсию

[текст лендинга]
```

## CoT для творческих задач

CoT работает не только для логики — он улучшает и творческие задачи:

```
Придумай концепцию онлайн-курса по AI для бизнеса.

Сначала подумай вслух:
- Кто целевая аудитория и какова их главная боль?
- Что они хотят получить в итоге?
- Почему существующие курсы их не устраивают?
- Какой формат обучения подойдёт лучше всего?

После анализа — предложи концепцию курса.
```

## Практическое задание

Возьми сложное решение которое ты откладываешь (бизнес-задача, выбор инструмента, стратегический вопрос). Напиши промпт с явной инструкцией думать пошагово. Добавь 4-5 шагов рассуждения. Сравни результат с тем что получилось бы без CoT.

## Тест

**Вопрос 1:** Для чего Chain-of-Thought особенно полезен?

A) Для быстрых простых ответов
B) Для многошаговых задач где важна логика рассуждений
C) Только для математических задач
D) Для генерации коротких текстов

**Вопрос 2:** Как активировать CoT самым простым способом?

A) Написать промпт на английском
B) Добавить "реши шаг за шагом" или "думай пошагово"
C) Увеличить длину промпта
D) Использовать специальные символы

**Вопрос 3:** Что происходит когда ИИ "прыгает" к ответу без рассуждений?

A) Ответ всегда правильный
B) Ответ быстрее — это плюс
C) Возрастает риск ошибки на сложных многошаговых задачах
D) Ничего особенного

**Вопрос 4:** CoT работает только для логических задач?

A) Да, только для математики и логики
B) Нет, он улучшает и творческие задачи через структурированный анализ
C) Да, для творчества нужны другие техники
D) CoT вообще не влияет на качество

**Вопрос 5:** Ты хочешь выбрать CRM-систему для команды. Как применить CoT?

A) "Какая CRM лучшая?"
B) "Думай пошагово: сначала определи наши требования, потом сравни варианты по каждому, потом дай рекомендацию"
C) CoT тут не нужен
D) "Назови все CRM системы"

---

### Ответы:
1. **B** — CoT критически важен для задач с несколькими шагами рассуждений. Без него модель может "угадывать" ответ вместо его вычисления.
2. **B** — "Думай пошагово" или "реши шаг за шагом" — простейшие активаторы CoT. Работают сразу.
3. **C** — Без промежуточных шагов модель может допустить ошибку в логике и уверенно выдать неправильный ответ.
4. **B** — CoT структурирует любое мышление. Для творческих задач он помогает сначала проанализировать контекст, потом генерировать идеи.
5. **B** — Структурированный CoT: требования → сравнение → рекомендация. Это превращает угадывание в обоснованный выбор.
